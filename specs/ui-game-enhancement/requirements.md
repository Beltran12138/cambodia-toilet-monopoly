# UI/UX 与游戏性增强需求文档

## 介绍

基于现有 THE WAY: 柬埔寨厕所大富翁游戏核心功能，进行 UI/UX 优化和游戏性增强，提升用户体验和可玩性。

## 需求

### 需求 1 - 视觉动画增强

**用户故事：** 作为玩家，我希望看到更流畅的动画效果，让游戏过程更加生动有趣。

#### 验收标准

1. When 玩家掷骰子时，骰子应当显示滚动动画（至少 0.5 秒），然后逐步显示两个骰子的结果
2. When 玩家移动时，玩家标记应当逐格移动到目标位置（每格 0.3 秒），而不是瞬间跳转
3. When 建造厕所时，应当显示建造动画（如锤子敲击效果 + 厕所图标渐入）
4. When HP 变化时，HP 仪表盘应当平滑过渡（0.5 秒渐变），低 HP 时闪烁警告
5. When 游戏状态改变（胜利/失败）时，应当有全屏庆祝/悲伤动画效果

### 需求 2 - 音效系统完善

**用户故事：** 作为玩家，我希望听到合适的音效反馈，增强游戏沉浸感。

#### 验收标准

1. When 掷骰子时，播放骰子滚动音效
2. When 移动玩家时，播放脚步音效
3. When 建造厕所时，播放建造音效
4. When HP 减少到危险值（<30）时，播放警告音效
5. When 触发随机事件时，播放事件卡片翻开的音效
6. When 游戏胜利/失败时，播放对应的庆祝/悲伤音乐
7. 提供音效开关控制（设置面板中）

### 需求 3 - 响应式布局优化

**用户故事：** 作为玩家，我希望在不同设备尺寸上都能正常游玩游戏。

#### 验收标准

1. While 在桌面端（1920x1080），游戏界面应当完整显示所有元素
2. While 在笔记本（1366x768），游戏界面应当自动缩放，保持可读性
3. While 在平板（768x1024），棋盘和控制面板应当垂直排列，适当简化装饰元素
4. While 在手机端（375x667），启用竖屏模式，棋盘缩小，控制面板改为底部抽屉式
5. When 窗口大小改变时，界面应当在 200ms 内自适应调整

### 需求 4 - 成就系统

**用户故事：** 作为玩家，我希望解锁成就来获得成就感和重复游玩的动力。

#### 验收标准

1. When 首次建造厕所时，解锁"第一座厕所"成就
2. When 单局建造 5 个厕所时，解锁"建筑大师"成就
3. When 村民以满 HP（150）获胜时，解锁"完美健康"成就
4. When 捐赠者单次捐款超过$800 时，解锁"慈善家"成就
5. When 连续 3 次掷出双数时，解锁"幸运之星"成就
6. When 村民在 HP<30 的情况下逆转获胜时，解锁"绝处逢生"成就
7. 成就应当在游戏结束时展示，并保存到本地存储
8. 提供成就查看界面（可在游戏中点击打开）

### 需求 5 - 更多随机事件

**用户故事：** 作为玩家，我希望体验更多样化的随机事件，增加游戏不确定性。

#### 验收标准

1. 新增至少 7 个随机事件，总数达到 20 个
2. 新增事件包括：
   - "名人代言"：获得名人支持，资金+300
   - "技术突破"：建造成本降低 30% 持续 2 回合
   - "村民互助"：村民 HP+12
   - "材料短缺"：下次建造成本翻倍
   - "媒体报导"：获得额外捐款$250
   - "恶劣天气"：村民 HP-8，所有捐赠者资金 -50
   - "落成典礼"：选择一个已建厕所，村民 HP+20
3. 每个事件必须有独特的图标、颜色和教育意义描述
4. 事件权重总和保持平衡，正面和负面事件比例约 60:40

### 需求 6 - 游戏难度选择

**用户故事：** 作为玩家，我希望选择不同难度级别来匹配我的技能水平。

#### 验收标准

1. When 开始新游戏时，提供难度选择界面（简单/普通/困难）
2. While 简单难度，村民初始 HP=120，厕所建造成本 -20%
3. While 普通难度，使用标准规则（当前设定）
4. While 困难难度，村民初始 HP=80，经过无厕所村庄 HP-20
5. 难度选择应当影响最终评分（困难难度通关获得更高评价）
6. 难度信息应当在教程中清晰说明

### 需求 7 - 数据统计面板

**用户故事：** 作为玩家，我希望在游戏结束后查看详细统计数据，了解我的表现。

#### 验收标准

1. When 游戏结束时，显示统计面板包含：
   - 总共建造的厕所数量（小型/大型分别统计）
   - 总捐款金额
   - 村民平均 HP
   - 使用的随机事件数量
   - 游戏时长
   - 最佳捐赠者（捐款最多）
2. 统计数据应当与历史最佳记录对比
3. 提供"分享成绩"按钮，生成可分享的图片或文字

### 需求 8 - 教程改进

**用户故事：** 作为新玩家，我希望通过交互式教程快速学会游戏规则。

#### 验收标准

1. When 首次开始游戏时，教程分步骤高亮显示关键 UI 元素
2. While 教程进行中，玩家只能操作当前步骤允许的按钮
3. 教程包含 5 个步骤：
   - 步骤 1：介绍游戏目标（村民 HP 和圈数）
   - 步骤 2：介绍村民角色和 HP 机制
   - 步骤 3：介绍捐赠者角色和建造机制
   - 步骤 4：演示一次完整的回合流程
   - 步骤 5：介绍胜利/失败条件
4. 提供"跳过教程"选项
5. 教程可随时从设置中重新打开
